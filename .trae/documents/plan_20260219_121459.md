# Plano de Modernização e Limpeza (Clean Code & Aesthetic Programming)

Para tornar o código profissional, limpo e corrigir os comandos defeituosos, executarei o seguinte plano em etapas rigorosas:

## 1. Refatoração do Core (Utils)
O arquivo `Utils/djs.js` é uma "gambiarra" de compatibilidade que não deve existir em um código profissional v14.
- **Ação:** Atualizar `Utils/commandKit.js` para usar `discord.js` nativo (v14) e remover a dependência de `djs.js`.
- **Ação:** Verificar e atualizar `Utils/interactions.js` e `Utils/embeds.js` se necessário.

## 2. Modernização de Comandos (Lote 1: Admin & Moderação)
Atualizar comandos para usar `EmbedBuilder` e `PermissionFlagsBits` (substituindo o antigo `Permissions.FLAGS`).
- **Arquivos:**
  - `Admin/embed.js`, `Admin/config_evento.js`
  - `Moderacao/ban.js`, `Moderacao/kick.js`, `Moderacao/clear.js`

## 3. Modernização de Comandos (Lote 2: Economia Restante)
Aplicar padrões de Clean Code (destructuring, early returns) e v14.
- **Arquivos:** `investigar.js`, `contrato.js`, `casar.js`, `bancocentral.js`, `retirar.js`, `depositar.js`, `crise.js`, `politica.js`, `extrato.js`, `rank.js`, `sobremim.js`.

## 4. Modernização de Comandos (Lote 3: Outros & Eventos)
- **Arquivos:** `eventohelp.js`, `sorteio.js`, `gif.js`, `ping.js`.
- **Eventos:** `Eventos/client/menção.js`, `Eventos/economia/electionAttractions.js`.

## 5. Limpeza Final
- **Ação:** Deletar o arquivo obsoleto `Utils/djs.js`.
- **Resultado:** O projeto ficará puramente em Discord.js v14, sem camadas de compatibilidade legada, mais rápido e fácil de manter.

Este plano garante que "defeitos" causados por mistura de versões sejam eliminados e o código siga padrões estéticos profissionais.
