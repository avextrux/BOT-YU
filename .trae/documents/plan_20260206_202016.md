**Plano de Correção e Otimização**

Vou realizar as seguintes melhorias para corrigir erros, modernizar o código e otimizar o desempenho:

1. **Otimização do Banco de Dados (`Database/contract.js`)**

   * **Problema:** A busca por `shortId` (ID curto) atualmente carrega os últimos 50 contratos e filtra em memória, o que é ineficiente e propenso a falhas.

   * **Solução:** Adicionar um campo `shortId` indexado no schema do Mongoose e um hook `pre('save')` para gerá-lo automaticamente. A busca passará a ser direta e rápida.

2. **Modernização e Otimização de Eventos (`Eventos/economia/contracts.js`)**

   * **Problema:** Loop sequencial (`for ... await`) para atualizar usuários, causando lentidão. Uso de sintaxe legada (`MessageEmbed`, `client.on("ready")`) e tratamento de erros fraco (`console.error`).

   * **Solução:**

     * Paralelizar as atualizações de usuários usando `Promise.all`.

     * Atualizar para `EmbedBuilder` (Discord.js v14).

     * Implementar `logger` estruturado.

3. **Modernização do Ticker do Mercado Negro (`Eventos/economia/blackMarketTicker.js`)**

   * **Problema:** Uso de `MessageEmbed` (legado) e `console.error`.

   * **Solução:** Atualizar para `EmbedBuilder` e implementar `logger`.

4. **Padronização Geral**

   * Garantir que os eventos usem `Discord.Events` em vez de strings mágicas.

**Sequência de Execução:**

1. Atualizar o Schema de Contratos.
2. Refatorar `contracts.js` (Eventos).
3. Refatorar `blackMarketTicker.js`.
